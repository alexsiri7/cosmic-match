## Codebase Patterns

### Flutter project structure
- **Where**: project root
- **Pattern**: Flutter project at repo root (not nested), standard `flutter create` layout
- **Example**: `lib/main.dart` → `CosmicMatchApp`, `lib/game/cosmic_match_game.dart` → `CosmicMatchGame extends FlameGame`

### Validation commands
- **Where**: CLI
- **Pattern**: Flutter SDK at `/tmp/flutter-sdk/bin`, use `export PATH="/tmp/flutter-sdk/bin:$PATH"` before commands
- **Example**: `flutter analyze`, `flutter test`

### Flame component patterns
- **Where**: `lib/game/components/`
- **Pattern**: Use `HasGameReference` mixin (not deprecated `HasGameRef`). Use `onGameResize` for responsive layout.
- **Example**: `class BoardComponent extends PositionComponent with HasGameReference`

### Tap handling in Flame
- **Where**: `lib/game/components/tile_component.dart`
- **Pattern**: Use `TapCallbacks` mixin on individual components (not `HasTappables` on game). Use `onTapUp` event handler.
- **Example**: `class TileComponent extends PositionComponent with TapCallbacks`

### Cascade pattern
- **Where**: `lib/game/components/board_component.dart:195-211`
- **Pattern**: `_startCascade()` async loop: clear → gravity → 300ms delay → detect → repeat. `cascadeCount` tracks chain depth. `_processing` stays true throughout.
- **Example**: `await _startCascade(matches);`

### Gravity pattern
- **Where**: `lib/models/board_state.dart`
- **Pattern**: `applyGravity()` returns `GravityResult` with `movedTiles` (map of new pos → old row) and `newTiles` (list of spawned positions). BoardComponent uses this to animate falls and spawn new TileComponents.
- **Example**: `final result = boardState.applyGravity(); for (final entry in result.movedTiles.entries) { ... }`

### Match detection pattern
- **Where**: `lib/utils/match_detector.dart`
- **Pattern**: MatchDetector.findMatches(board) scans rows then columns for consecutive same-type runs, skipping nulls and obstacle tiles. Overlapping matches are merged via set intersection for L/T shapes.
- **Example**: `final matches = MatchDetector.findMatches(board);`

### Swap animation pattern
- **Where**: `lib/game/components/board_component.dart`
- **Pattern**: Use Flame `MoveEffect.to()` with `EffectController(duration: 0.2)` for tile swap animation. Use `onComplete` callback on the second effect to handle post-swap logic.
- **Example**: `tileA.add(MoveEffect.to(posB, EffectController(duration: 0.2)))`
